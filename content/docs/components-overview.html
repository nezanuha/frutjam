Frutjam includes 48 production-ready components organized by category. This guide covers component organization, naming conventions, composition patterns, and how to choose the right component for your use case.

Component Organization

Components are organized into these functional categories:

Form Components (8)

User input and form controls:

  • Input - Text input fields
  • Textarea - Multi-line text input
  • Select - Dropdown selection
  • Checkbox - Multiple selection
  • Radio - Single selection
  • Toggle - Binary on/off switch
  • Range - Slider for numeric values
  • Combobox - Searchable select input

Navigation Components (7)

Structural navigation and menus:

  • Navbar - Top horizontal navigation
  • Menu - Vertical action list
  • Breadcrumb - Navigation hierarchy
  • Tabs - Content tabs
  • Steps - Sequential progress
  • Sidebar - Side panel navigation
  • Header - Page header with branding

Content Components (13)

Display and organize content:

  • Card - Content container
  • Badge - Small label
  • Tag - Categorization tag
  • Avatar - User profile image
  • Divider - Visual separator
  • Table - Data grid
  • Chat - Message bubbles
  • Hero - Large banner section
  • Footer - Page footer
  • Carousel - Image/content slider
  • Timeline - Chronological display
  • Stat - Metric display
  • Indicator - Status badge overlay

Interactive Components (10)

User interaction and modals:

  • Button - Clickable action
  • Link - Styled hyperlink
  • Modal - Dialog overlay
  • Drawer - Side slide panel
  • Popover - Floating content
  • Tooltip - Context information
  • Collapsible - Expandable section
  • Swap - Toggle between states
  • Join - Grouped adjacent items
  • Accordion - Multiple collapse sections

Feedback Components (7)

User feedback and status:

  • Alert - Information/warning/error message
  • Progress - Progress bar
  • Radial Progress - Circular progress
  • Loading - Activity indicator
  • Rating - Star rating
  • Status - User/system status indicator
  • Countdown - Time remaining

Utility Components (3)

Layout and presentation:

  • Surface - Layered container
  • Mask - Shape masking
  • Diff - Side-by-side comparison

Text Components (1)

Typography utilities:

  • Keyboard (KBD) - Keyboard key display

Naming Conventions

All components follow consistent naming patterns:

Base Classes

  • .btn - Base component (button)
  • .card - Base component (card)
  • .input - Base component (input)
  • .modal - Base component (modal)

Variants

  • .btn-primary - Color variant
  • .btn-lg - Size variant
  • .btn-outline - Style variant
  • .btn-circle - Shape variant
  • .btn-disabled - State variant

Modifiers

  • .btn-group - Button grouping
  • .menu-item - Menu item subcomponent
  • .card-title - Card title subcomponent
  • .alert-title - Alert title subcomponent

Component Dependencies

Most Frutjam components are independent. However, some combine well:

  • Button + Icon - Icon buttons with tooltips
  • Card + Badge - Cards with status badges
  • Alert + Button - Actionable alerts
  • Modal + Form - Confirmation dialogs
  • Menu + Avatar - User dropdown menus

Component States

Most components support these state variations:

  • Default - Base appearance
  • Hover - Mouse over (desktop)
  • Active - Currently selected or pressed
  • Disabled - Inactive and unusable
  • Loading - Async operation in progress

Accessibility by Component

All Frutjam components include:

  • Semantic HTML
  • ARIA attributes where needed
  • Keyboard navigation support
  • Focus indicators
  • Color contrast (WCAG AA)
  • Screen reader compatibility
