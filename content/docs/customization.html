Frutjam is built to be customized. Extend components, override styles, create custom utilities, and integrate with your design system. This guide covers extending Frutjam without modifying its core files.

CSS Custom Properties (Variables)

All Frutjam components use CSS custom properties, making them easy to customize globally or per-component:

Common Custom Properties

/* Border radius */ --border-radius: 0.25rem; /* Spacing scale */ --spacing-xs: 0.25rem; --spacing-sm: 0.5rem; --spacing-md: 1rem; /* Colors (see Colors & Tokens page) */ --color-primary: oklch(51.1% 0.262 276.966); --color-on-primary: oklch(96.2% 0.018 272.314);

Customizing Global Variables

/* In your input.css or global styles */ @layer base { :root { --border-radius: 0.5rem; /* More rounded components */ --color-primary: oklch(40% 0.25 280); /* Custom primary */ } }

Per-Component Customization

/* Override button border radius only */ .btn { border-radius: 9999px; /* Pill-shaped buttons */ } /* Custom button variant */ .btn-custom { background-color: var(--color-primary); border: 2px dashed var(--color-primary); }

Extending Components

Add New Component Variants

/* Add a new button size */ @layer components { .btn-xlg { @apply px-8 py-4 text-xl; } .btn-xlg.btn-primary { @apply bg-primary hover:bg-primary-dark; } }

Creating Custom Utilities

Add custom utility classes using Tailwind's layer system:

Simple Utility

@layer utilities { .text-balance { text-wrap: balance; } .truncate-lines-2 { display: -webkit-box; -webkit-line-clamp: 2; -webkit-box-orient: vertical; overflow: hidden; } }

Utility with Variants

@layer utilities { .blur-sm { filter: blur(4px); } @variant hover { .blur-sm:hover { filter: blur(2px); } } }

Theme Configuration

Customize Tailwind's theme configuration while keeping Frutjam styles:

Extending Colors

/* In your tailwind.config.js or CSS */ @theme { colors: { custom: { 50: oklch(98% 0.01 280), 500: oklch(70% 0.15 280), 900: oklch(20% 0.05 280), } } }

Custom Breakpoints

@theme { screens: { 'sm': '640px', 'md': '768px', 'lg': '1024px', 'xl': '1280px', '2xl': '1536px', 'tablet': '768px', 'desktop': '1024px', } }

Component-Specific Overrides

Override Button Styles

.btn { /* Custom transitions */ transition: all 300ms cubic-bezier(0.4, 0, 0.2, 1); /* Custom shadow */ box-shadow: 0 4px 6px rgba(0, 0, 0, 0.1); } .btn:hover { box-shadow: 0 6px 12px rgba(0, 0, 0, 0.15); }

Override Modal Backdrop

.modal-backdrop { background-color: rgba(0, 0, 0, 0.7); /* Darker backdrop */ backdrop-filter: blur(4px); }

Scoped Styling

Apply styles to specific sections without affecting components globally:

/* Custom theme for a specific section */ .dark-section { --color-base: oklch(0.15 0 0); --color-on-base: oklch(1 0 0); --color-primary: oklch(60% 0.25 280); }

Disabling Components

If you don't need certain components, exclude them for smaller bundle size:

/* In your input.css, comment out unneeded imports */ @import "tailwindcss"; @import "frutjam/components/button"; @import "frutjam/components/card"; /* @import "frutjam/components/carousel"; */ /* Not needed */

Best Practices for Customization

  • Use CSS custom properties instead of hardcoded values
  • Extend rather than override - maintain consistency
  • Test responsive behavior when customizing
  • Document custom variants for team consistency
  • Keep accessibility standards when styling
  • Use semantic class names for custom utilities
  • Avoid using !important - fix specificity instead
