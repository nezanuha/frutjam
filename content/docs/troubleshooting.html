Common issues and solutions when using Frutjam. If you encounter problems, check this guide first. For additional help, see the FAQ section or file an issue on GitHub.

Installation Issues

Styles Not Loading

Problem: Components appear unstyled.

Solutions:

  • Verify Frutjam is imported in your CSS file: @import "tailwindcss"; @import "frutjam";
  • Check file path is correct (case-sensitive on Linux)
  • Ensure Tailwind CSS is installed: npm list tailwindcss
  • Rebuild your CSS after adding import
  • Clear browser cache (Ctrl+Shift+Delete)

NPM Installation Fails

Problem: Error when running npm install frutjam

Solutions:

  • Clear npm cache: npm cache clean --force
  • Delete node_modules and package-lock.json, then reinstall
  • Check Node.js version: node --version (requires Node 14+)
  • Try installing from latest version: npm install frutjam@latest
  • Check internet connection and npm registry access

Component Issues

Button Styles Not Showing

Problem: Buttons appear unstyled or default.

Solutions:

  • Verify class name: should be .btn, not .button
  • Check CSS is imported correctly
  • Inspect element to see if classes are applied
  • Check for conflicting CSS rules (use specificity) /* Check in DevTools if this override exists */ button { all: unset; /* This removes all button styles */ }

Modal Not Opening

Problem: Modal doesn't appear when clicking trigger.

Solutions:

  • Verify modal has unique ID and button has matching popovertarget:
    Content
  • Check dialog element has open attribute or JavaScript handler
  • Verify JavaScript isn't preventing event propagation
  • Check browser supports native dialog/popover API

Dropdown Menu Positioning Wrong

Problem: Dropdown appears in wrong location.

Solutions:

  • Check parent element has position: relative or uses dropdown-end:
    • Item
  • Verify parent container isn't overflow: hidden
  • Check z-index isn't being overridden

Styling Issues

Colors Don't Match Theme

Problem: Component colors are different from expected.

Solutions:

  • Verify theme is applied correctly:
  • Check custom CSS doesn't override component colors
  • Inspect element to see actual CSS variables being used
  • Verify theme CSS variables are defined

Responsive Classes Not Working

Problem: md:, lg: prefixes don't work.

Solutions:

  • Ensure you're using correct Tailwind breakpoint syntax
  • Common breakpoints:
    Mobile: 100% width Tablet+: 50% width Desktop+: 33% width
  • Check in DevTools that responsive styles are generated
  • Verify build tool is configured for responsive classes

Custom Styles Not Applied

Problem: Custom CSS doesn't override component styles.

Solutions:

  • Use correct CSS specificity - component classes use classes, not IDs: /* This works */ .btn.custom-btn { background-color: red; } /* This also works, but more specific */ button.btn.custom-btn { background-color: red; } /* Avoid !important unless necessary */
  • Ensure custom styles are loaded AFTER Frutjam styles
  • Check styles aren't scoped/isolated from components

Accessibility Issues

Keyboard Navigation Not Working

Problem: Can't navigate with Tab key.

Solutions:

  • Verify interactive elements are native buttons/inputs:
    Click me
  • Add tabindex="0" to custom interactive elements
  • Check JavaScript doesn't prevent default keyboard events
  • Verify focus indicators are visible

Screen Reader Doesn't Announce Content

Problem: Screen reader skips elements.

Solutions:

  • Add ARIA labels to icon-only buttons:
  • Use semantic HTML elements (button, input, select)
  • Check for hidden content: display: none hides from screen readers
  • Test with actual screen reader (NVDA, JAWS, VoiceOver)

Performance Issues

CSS File Too Large

Problem: Large CSS bundle size.

Solutions:

  • Only import components you use: /* Instead of importing all */ @import "frutjam"; /* Import only what you need */ @import "frutjam/components/button"; @import "frutjam/components/card";
  • Enable PurgeCSS/Tree Shaking in build
  • Minify CSS in production
  • Use compression (gzip/brotli)

Page Loads Slowly

Problem: Slow initial load time.

Solutions:

  • Check CSS file size and load method
  • Load CSS synchronously before HTML renders
  • Defer JavaScript that isn't critical
  • Use DevTools Network tab to identify slow resources
  • Optimize images separately from CSS

FAQ

Q: Can I use Frutjam with other CSS frameworks?

A: Not recommended. Frutjam is built on Tailwind CSS, and combining with other frameworks often causes conflicts. If needed, use CSS namespacing (prefixes) for isolation.

Q: How do I create custom components?

A: Use Tailwind utilities and Frutjam components as building blocks. Combine them using Flexbox and Grid. See the Components Overview and Examples pages.

Q: Can I change component colors?

A: Yes! Use theme system to customize colors. See the Themes page.

Q: Does Frutjam work with React/Vue/etc?

A: Yes! Frutjam is framework-agnostic - just CSS and HTML. Works with any JavaScript framework.

Q: How do I update Frutjam?

A: Run npm update frutjam or npm install frutjam@latest

Q: What about older browsers?

A: See Browser Support page for compatibility and fallback options.

Q: Where do I report bugs?

A: File issues on GitHub Issues with browser version and reproduction steps.
