Frutjam uses a mobile-first, breakpoint-based approach to responsive design. Define your mobile layout first, then enhance it for larger screens using Tailwind's responsive prefixes. This guide covers breakpoints, strategies, and best practices.

Default Breakpoints

Frutjam inherits Tailwind CSS v4's responsive breakpoints:

Name Breakpoint Min Width CSS
None (mobile) - 0px (default)
sm @sm 640px @screen sm
md @md 768px @screen md
lg @lg 1024px @screen lg
xl @xl 1280px @screen xl
2xl @2xl 1536px @screen 2xl

Container Queries (CSS)

For component-level responsiveness independent of viewport:

/* Define container */ .card-container { container-type: inline-size; } /* Query container size */ @container (min-width: 400px) { .card-title { font-size: 1.5rem; } }

Testing Responsive Design

Browser DevTools

  • Chrome: Ctrl+Shift+M (or Cmd+Shift+M on Mac)
  • Firefox: Ctrl+Shift+M (or Cmd+Shift+M on Mac)
  • Safari: Cmd+Shift+R (responsive design mode)

Test Common Sizes

  • Mobile: 375px (iPhone SE), 414px (iPhone 13), 390px (Pixel 6)
  • Tablet: 768px (iPad), 810px (iPad Air)
  • Desktop: 1024px, 1280px, 1536px+

Recommended Testing

  • Test at min/max breakpoints (640px, 1024px, etc.)
  • Test on actual devices when possible
  • Verify touch targets are 44px+ on mobile
  • Test landscape orientation on mobile
  • Test with text zoom up to 200%

Performance Considerations

  • Use Tailwind's responsive prefixes (minimal overhead)
  • Avoid loading large images on mobile
  • Use media queries for heavy CSS changes
  • Test performance on slow networks (DevTools throttling)
  • Optimize images for different screen densities

Responsive Design Checklist

  • ☐ Mobile layout defined first (no breakpoint prefix)
  • ☐ Tested on mobile, tablet, and desktop
  • ☐ Touch targets are minimum 44px × 44px
  • ☐ Images are responsive and optimized
  • ☐ Text remains readable at all sizes
  • ☐ Landscape orientation works properly
  • ☐ No horizontal scrolling on mobile
  • ☐ Forms are usable on touch devices
  • ☐ Performance is acceptable on mobile networks
