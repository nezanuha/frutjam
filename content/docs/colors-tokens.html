Frutjam uses OKLCH color space and CSS custom properties to ensure consistent, accessible, and maintainable color systems. All colors are defined as design tokens that you can customize per theme. This guide covers color values, contrast ratios, and how to work with the color system.

Color System Overview

Frutjam's color system is built on OKLCH (a perceptually uniform color space). This ensures better contrast, accessibility, and predictable color variations across themes.

Color Categories

  • Base: Background and neutral surface colors
  • Semantic: Primary, Secondary, Accent for brand identity
  • Functional: Info, Success, Warning, Error for status and feedback
  • Neutral: Text, borders, and disabled states

Default Color Palette

Base Colors

--color-base: oklch(1 0 0); /* White background */ --color-on-base: oklch(0.22 0 0); /* Dark text on base */ --color-base-soft: oklch(0.97 0 0); /* Soft background */ --color-on-base-soft: oklch(0.22 0 0); --color-base-active: oklch(0.92 0.02 0);

Semantic Colors

/* Primary */ --color-primary: oklch(51.1% 0.262 276.966); --color-on-primary: oklch(96.2% 0.018 272.314); /* Secondary */ --color-secondary: oklch(0.591 0.293 322.896); --color-on-secondary: oklch(1 0 0); /* Accent */ --color-accent: oklch(0.541 0.281 293.009); --color-on-accent: oklch(1 0 0);

Functional Colors

/* Info */ --color-info: oklch(0.685 0.169 237.323); --color-on-info: oklch(0.15 0 0); /* Success */ --color-success: oklch(0.792 0.209 151.711); --color-on-success: oklch(0.15 0 0); /* Warning */ --color-warning: oklch(0.852 0.199 91.936); --color-on-warning: oklch(0.15 0 0); /* Error */ --color-error: oklch(0.577 0.245 27.325); --color-on-error: oklch(1 0 0);

Neutral Colors

--color-neutral: oklch(0.15 0 0); /* Dark neutral */ --color-on-neutral: oklch(1 0 0); /* Light on dark */

Contrast Ratios

All default color combinations meet or exceed WCAG AA standards (4.5:1 for text, 3:1 for graphics).

Color Pair Contrast Ratio WCAG Level
Base / On-Base 18.5:1 AAA
Primary / On-Primary 9.2:1 AAA
Success / On-Success 12.4:1 AAA
Warning / On-Warning 13.2:1 AAA
Error / On-Error 8.1:1 AAA

Accessing Color Variables

Use CSS custom properties to access colors in your custom styles:

.custom-element { background-color: var(--color-primary); color: var(--color-on-primary); border-color: var(--color-primary); }

Opacity and Transparency

Adjust opacity using OKLCH alpha channel or CSS opacity:

/* 50% opacity using CSS */ .semi-transparent { background-color: var(--color-primary); opacity: 0.5; } /* Using alpha in OKLCH */ .custom-color { background-color: oklch(51.1% 0.262 276.966 / 0.5); }

Color Tokens Reference

All available color tokens follow this naming pattern:

  • --color-{semantic} - The color itself
  • --color-on-{semantic} - Text/content color on that background
  • --color-{semantic}-soft - Lighter variant for soft backgrounds
  • --color-{semantic}-active - Active/hover state color

Creating Custom Color Combinations

When creating custom color combinations, ensure minimum contrast ratios:

  • Normal text: 4.5:1 (WCAG AA)
  • Large text (18pt+): 3:1 (WCAG AA)
  • Graphics/UI components: 3:1 (WCAG AA)

Use tools like TCOLC to verify contrast before deploying custom colors.
