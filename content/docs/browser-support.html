CSS Features Required

Frutjam relies on these modern CSS features. Ensure your target browsers support them:

Essential Features

  • CSS Grid - For layout components
  • Flexbox - For flexible layouts
  • CSS Custom Properties - For theming and customization
  • CSS Transitions - For smooth animations
  • CSS Transforms - For visual effects

Advanced Features

  • OKLCH Color Space - For color calculations
  • CSS Container Queries - For responsive components (optional)
  • :is() Selector - For selector specificity (fallback: standard selectors)
  • CSS Mask - For mask component

Older Browser Fallbacks

If you need to support older browsers, consider these alternatives:

CSS Grid to Flexbox Fallback

/* Older browsers: use flexbox */ .card-container { display: flex; flex-wrap: wrap; } /* Modern browsers: use grid */ @supports (display: grid) { .card-container { display: grid; grid-template-columns: repeat(auto-fit, minmax(300px, 1fr)); } }

CSS Variables Fallback

.button { /* Fallback for older browsers */ background-color: #5B21B6; color: #ffffff; /* Modern with CSS variables */ background-color: var(--color-primary, #5B21B6); color: var(--color-on-primary, #ffffff); }

OKLCH to RGB Fallback

.primary { /* RGB fallback */ background-color: rgb(91, 33, 182); /* Modern OKLCH */ background-color: oklch(51.1% 0.262 276.966); }

Common Browser Issues

Safari Issues

  • Sticky positioning: Limited in older versions - use position: fixed as fallback
  • Custom properties: Full support in Safari 9.1+
  • Grid gaps: Full support in Safari 10.1+
  • appearance property: Use -webkit prefix for full support

Firefox Issues

  • Grid - Full support since Firefox 52
  • Flexbox - Full support since Firefox 48
  • Custom Properties - Full support since Firefox 31
  • OKLCH Colors - Supported since Firefox 113

Older Edge (Pre-Chromium)

  • Limited CSS custom property support
  • Incomplete Grid implementation
  • Recommend upgrading to Chromium-based Edge

Polyfills

If you must support very old browsers, consider these polyfills:

  • CSS Vars Polyfill - For custom properties (older IE)
  • PostCSS - Transform modern CSS for older browsers
  • Autoprefixer - Add vendor prefixes automatically

Example PostCSS Configuration

// postcss.config.js module.exports = { plugins: { autoprefixer: {}, 'postcss-preset-env': { stage: 3, features: { 'oklch-function': false // Exclude OKLCH if not needed } } } };

Performance Considerations

Browser Caching

  • Set appropriate cache headers for CSS files
  • Use long-term caching for versioned assets
  • Leverage browser cache for repeated visits

Download Size

  • Minify CSS in production
  • Use CSS compression (gzip, brotli)
  • Consider critical CSS inlining for above-the-fold

Accessibility Across Browsers

  • All modern browsers support ARIA attributes
  • Test screen readers (NVDA, JAWS, VoiceOver)
  • Verify keyboard navigation works
  • Check focus indicators are visible

Browser Support Checklist

  • ☐ Tested on latest Chrome, Firefox, Safari, Edge
  • ☐ Tested on iOS Safari and Chrome Android
  • ☐ Tested in older browser versions (if required)
  • ☐ Verified CSS features are supported
  • ☐ Fallbacks in place for unsupported features
  • ☐ Responsive design tested on multiple sizes
  • ☐ Touch interactions work on mobile
  • ☐ Accessibility works across browsers
